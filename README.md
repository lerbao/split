# ADB 自动化脚本集

通过 ADB 控制 Android 设备，支持自动滑动和抖音互动。

## 前置条件

1. 安装 [ADB](https://developer.android.com/tools/adb) 并加入 PATH
2. Android 设备开启 USB 调试，通过 `adb devices` 确认已授权

---

## 1. swipe_loop.ps1 — 通用自动滑动

纯滑动脚本，适用于任何需要上下滑动的 App。

```powershell
.\swipe_loop.ps1
```

### 参数说明

| 参数 | 随机范围 | 说明 |
|------|----------|------|
| 起始 X | 480 ~ 599 | 起始横坐标 |
| 起始 Y | 1900 ~ 2299 | 起始纵坐标（屏幕底部） |
| 终点 X | 480 ~ 599 | 终点横坐标 |
| 终点 Y | 100 ~ 499 | 终点纵坐标（屏幕顶部） |
| 持续时间 | 300 ~ 799ms | 滑动动作耗时 |
| 间隔 | 10/11/12/13/14/15s | 每次滑动后等待时间 |

## 2. douyin_bot.ps1 — 抖音自动刷视频 Bot

在抖音界面自动刷视频，并随机执行互动操作。

```powershell
.\douyin_bot.ps1
```

### 动作概率

| 动作 | 概率 | 说明 |
|------|------|------|
| 点赞 | 40% | 双击屏幕中央红心 |
| 收藏 | 15% | 点击右侧收藏图标 |
| 评论 | 10% | 打开评论→输入随机文字→发送 |
| 跳过 | 35% | 仅滑动，不互动 |

### 抖音按钮坐标（1080x2388 基准，带随机偏移）

| 按钮 | 基准坐标 | 偏移范围 |
|------|----------|----------|
| 心形(点赞) | (930, 1350) | ±30px |
| 评论图标 | (930, 1580) | ±30px |
| 收藏图标 | (930, 1820) | ±30px |
| 双击点赞 | (540, 1200) | ±100px |
| 评论输入框 | (540, 2280) | ±100px |
| 发送按钮 | (980, 2280) | ±30px |

> **评论功能**：需要安装 [ADBKeyboard](https://github.com/senzhk/ADBKeyBoard) apk，脚本通过 `adb shell am broadcast -a ADB_INPUT_TEXT` 广播方式输入中文。

## 3. douyin_bot.js — Auto.js 抖音自动滑动（手机端独立运行）

无需连接电脑，在手机上通过 Auto.js 独立运行，自动滑动抖音视频。

### 所需文件

| 文件 | 路径 | 说明 |
|------|------|------|
| `douyin_bot.js` | 本项目 | Auto.js 滑动脚本 |
| Auto.js APK | 安装到手机 | 运行环境，[Auto.js6 下载](https://github.com/SuperMonster002/AutoJs6) |

### 部署步骤

1. **安装 Auto.js**：在手机上安装 Auto.js6 APK
2. **开启无障碍服务**：系统设置 → 无障碍 → 找到 Auto.js → 开启
3. **推送脚本到手机**：
   ```powershell
   adb push "d:\workspace\trae\滑动\douyin_bot.js" /storage/emulated/0/脚本/
   ```
4. **打开抖音**：确保在抖音短视频播放界面
5. **运行脚本**：在 Auto.js 中导入并运行 `douyin_bot.js`

### 脚本行为

| 参数 | 值 | 说明 |
|------|-----|------|
| 滑动 X 坐标 | 100 ~ 900 随机 | 模拟手指不同位置 |
| 起始 Y | 1600 | 屏幕中下部 |
| 终点 Y | 400 | 屏幕上部 |
| 滑动耗时 | 300 ~ 800ms | 模拟自然滑动速度 |
| 滑动间隔 | 5 ~ 10s | 模拟观看时间 |

### 停止方式

按**音量 - 键**即可停止，屏幕会弹出"滑动已停止"提示。

### 注意事项

1. **无障碍服务必须开启**，否则 `swipe()` 无法执行
2. **手机需保持亮屏解锁状态**，熄屏后滑动失效
3. **不插 SIM 卡也能用**，连 WiFi 即可，滑动是本地触屏模拟
4. **无需 root 权限**，Auto.js 基于无障碍服务实现
5. **首次运行**会弹出"无障碍服务"和"悬浮窗"权限请求，全部允许
6. **Y 坐标适配**：当前 1600→400 基于 1080×2400 分辨率，如果手机分辨率不同需调整
7. **不要锁屏运行**，熄屏后脚本会继续跑但滑动无效，建议开"保持屏幕常亮"
8. **抖音版本兼容**：只要抖音是上下滑动切换视频的交互，脚本就有效

## ADB 常用命令

```
adb shell input swipe <x1> <y1> <x2> <y2> [duration(ms)]
```

- `x1, y1` — 起始坐标
- `x2, y2` — 终点坐标
- `duration` — 滑动持续时间，单位毫秒
